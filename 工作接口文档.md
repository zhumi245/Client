| 方向 | 接口 | 传输方式 | 频率 | 当前状态 |
|:----:|:----|:--------:|:----:|:--------:|

## 2. 服务端 → 客户端（数据推送）

### 2.1 TCP连接

| 项 | 值 |
|:---|:----|
| 协议 | TCP/IP |
| 服务端地址 | 0.0.0.0 |
| 端口 | 5000 |
| 客户端地址 | 127.0.0.1 |
| 消息分隔符 | `\n`（换行符） |
| 编码 | UTF-8 |
| 连接超时 | 5秒 |
| 自动重连 | 3秒间隔 |

### 2.2 JSON数据格式

服务端每100ms广播一次JSON，示例:
```json
{
  "speed": 98.5,
  "rpm": 3940,
  "fuel": 59.45,
  "temp": 88.3,
  "turnSignal": 1,
  "hazard": 0,
  "absFault": 0,
  "airbagFault": 0,
  "batteryFault": 0,
  "brakeFault": 0,
  "speedFault": 0
}
```

### 2.3 字段定义

| 字段 | 类型 | 范围 | 精度 | 说明 | 客户端Mock |
|:----|:----|:----:|:----:|:-----|:---------:|
| speed | double | 0~140 | 1位小数 | 速度(km/h) | ✅ 正弦波60±80，周期10s |
| rpm | int | 0~8000 | - | 转速(RPM) = speed×40 | ✅ 自动计算 |
| fuel | double | 0~60 | 2位小数 | 油量(L) | ✅ 递减0.001L/100ms，初始60 |
| temp | double | 40~120 | 1位小数 | 水温(°C) | ✅ 随机游走80~100±5°C |
| turnSignal | int | 0/1/2 | - | 0=关 1=左 2=右 | ✅ 菜单控制 |
| hazard | int | 0/1 | - | 0=关 1=开 | ✅ 菜单控制 |
| absFault | int | 0/1 | - | ABS故障 | ✅ 3%/s随机，5~15s恢复 |
| airbagFault | int | 0/1 | - | 安全气囊故障 | ✅ 3%/s随机，5~15s恢复 |
| batteryFault | int | 0/1 | - | 电瓶故障 | ✅ 3%/s随机，5~15s恢复 |
| brakeFault | int | 0/1 | - | 刹车故障 | ✅ 3%/s随机，5~15s恢复 |
| speedFault | int | 0/1 | - | 超速报警(speed>120) | ✅ speed>120自动触发 |

**字段存在性**: 客户端对每个字段做 `value().isUndefined()` 检查，缺失字段使用默认值不崩溃。

**值钳位**: speed≥0, rpm≥0, fuel≥0, temp在40~120之间。

### 2.4 故障触发逻辑（客户端内建）

以下报警由客户端内建逻辑自动触发（不依赖服务端JSON字段）:

| 报警索引 | 标签 | 触发条件 |
|:-------:|:----:|:---------|
| 3 | TEMP | 客户端 temp > 95°C 时自动触发 |
| 4 | FUEL | 客户端 fuel < 15L 时自动触发 |

## 3. 客户端 → 服务端（控制指令）

### 3.1 指令格式

每条指令以换行符 `\n` 结束，纯文本格式（非JSON）。

### 3.2 指令列表

| 指令 | 格式示例 | 说明 | 客户端Mock |
|:----|:---------|:-----|:---------:|
| 左转 | `turnSignal:1\n` | 打开左转向灯 | ✅ `sendTurnSignal(1)` |
| 右转 | `turnSignal:2\n` | 打开右转向灯 | ✅ `sendTurnSignal(2)` |
| 关转向 | `turnSignal:0\n` | 关闭转向灯 | ✅ `sendTurnSignal(0)` |
| 双闪开 | `hazard:1\n` | 打开双闪警告 | ✅ `sendHazard(true)` |
| 双闪关 | `hazard:0\n` | 关闭双闪警告 | ✅ `sendHazard(false)` |
| 重置 | `reset\n` | 重置所有状态到初始值 | ✅ `reset()` |

### 3.3 互斥逻辑

- 左转和右转互斥（选择左转自动取消右转）
- 双闪(hazard)覆盖转向灯状态（hazard=1时忽略turnSignal）

## 4. 客户端内部接口（IDataSource抽象层）

### 4.1 架构设计

```
IDataSource (抽象接口)
  ├── MockDataSource  ← 当前默认模式
  └── NetworkClient   ← 联调模式 (--connect)
  
MainWindow → 面向IDataSource编程，不依赖具体实现
```

### 4.2 IDataSource接口定义

**文件**: `dashboard/idatasource.h`

**纯虚函数**:
| 函数 | 参数 | 说明 |
|:-----|:-----|:------|
| sendTurnSignal(int) | 0/1/2 | 发送转向灯指令 |
| sendHazard(bool) | true/false | 发送双闪指令 |
| reset() | - | 重置所有状态 |
| start() | - | 启动数据源 |
| stop() | - | 停止数据源 |

**Qt信号**:
| 信号 | 参数 | 说明 |
|:-----|:-----|:------|
| speedUpdated(double) | speed | 速度更新 |
| rpmUpdated(int) | rpm | 转速更新 |
| fuelUpdated(double) | fuel | 油量更新 |
| tempUpdated(double) | temp | 水温更新 |
| turnSignalUpdated(int) | 0/1/2 | 转向灯状态更新 |
| hazardUpdated(bool) | true/false | 双闪状态更新 |
| faultUpdated(int,bool) | index,active | 报警灯状态更新 (index 0-7) |
| connected() | - | 已连接 |
| disconnected() | - | 已断开 |
| statusMessage(QString) | msg | 状态栏信息 |

### 4.3 报警灯索引映射

| 索引 | 枚举 | 标签 | JSON字段 | Mock自动触发 |
|:---:|:----|:----:|:---------|:-----------:|
| 0 | Warning_Engine | ENG | speedFault | ✅ speed>120 |
| 1 | Warning_Oil | OIL | (手动) | ❌ 仅菜单 |
| 2 | Warning_Battery | BAT | batteryFault | ✅ 3%/s随机 |
| 3 | Warning_Temp | TEMP | temp>95°C | ✅ 内建逻辑 |
| 4 | Warning_Fuel | FUEL | fuel<15L | ✅ 内建逻辑 |
| 5 | Warning_Brake | BRAKE | brakeFault | ✅ 3%/s随机 |
| 6 | Warning_Airbag | AIR | airbagFault | ✅ 3%/s随机 |
| 7 | Warning_ABS | ABS | absFault | ✅ 3%/s随机 |

### 4.4 运行模式切换

| 模式 | 命令 | 数据源 |
|:----|:-----|:-------|
| Mock模式(默认) | `dashboard.exe` | MockDataSource |
| 网络模式 | `dashboard.exe --connect` | NetworkClient |

## 5. Mock说明

### 5.1 Mock覆盖范围

所有接口均已覆盖Mock实现，无遗漏:
- ✅ 所有11个JSON字段的模拟生成
- ✅ 所有6个控制指令的模拟响应
- ✅ 5种随机故障模拟
- ✅ 超速自动故障
- ✅ 油量/水温阈值报警

### 5.2 Mock局限

- MockDataSource只用于前端自测，不覆盖P6模拟器的全部功能
- 故障概率（3%/秒）是简化版，P6模拟器可能有更精确的故障模型
- 联调时必须切换到 NetworkClient 模式
- P5未实现的报警灯(OIL)只能通过菜单手动触发

### 5.3 联调切换步骤

1. P6启动模拟器（监听5000端口）
2. 运行：`dashboard.exe --connect`
3. 观察状态栏显示"Connected to simulator"
4. 验证所有仪表数据来自模拟器而非Mock
