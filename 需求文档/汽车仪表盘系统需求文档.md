# 汽车仪表盘系统需求文档

## 1. 项目概述

### 1.1 项目名称
汽车仪表盘系统（Car Dashboard System）

### 1.2 项目组成
本系统由两个独立应用程序组成：

| 应用程序 | 描述 | 通信方式 |
|---------|------|---------|
| `final_simulator` | 汽车信号模拟器（TCP服务器） | TCP Server，监听端口5000 |
| `dashboard` | 汽车仪表盘显示程序（TCP客户端） | TCP Client，连接localhost:5000 |

### 1.3 系统架构

```
┌─────────────────────┐         JSON over TCP          ┌─────────────────────┐
│  final_simulator    │ ──────────────────────────────▶  │     dashboard       │
│  (信号模拟器)        │ ◀──────────────────────────────  │    (仪表盘显示)      │
│                     │      控制指令 (turnSignal,       │                     │
│  - 速度             │       hazard, reset)             │  - 速度表           │
│  - 转速(RPM)        │                                   │  - 转速表           │
│  - 油量             │                                   │  - 油量表           │
│  - 水温             │                                   │  - 水温表           │
│  - 转向灯           │                                   │  - 转向灯           │
│  - 报警灯           │                                   │  - 报警灯组         │
└─────────────────────┘                                   └─────────────────────┘
```

### 1.4 技术栈
- **框架**: Qt 5.x
- **语言**: C++
- **通信**: TCP/IP (QTcpServer / QTcpSocket)
- **数据格式**: JSON
- **GUI**: QWidget + QPainter 自绘仪表

## 2. 功能需求

### 2.1 信号模拟器 (final_simulator)

#### 2.1.1 TCP服务器
- 监听端口: **5000**
- 支持多客户端连接
- 每100ms向所有已连接客户端发送一次JSON数据
- 接收客户端控制指令并响应

#### 2.1.2 信号生成

| 信号名 | 生成规则 | 数据类型 | 单位 |
|-------|---------|---------|------|
| speed | 正弦波: 60 ± 80 km/h，周期10秒，范围0~140 | double | km/h |
| rpm | rpm = speed × 40 | int | RPM |
| fuel | 初始60L，每100ms减少0.001L，最小0 | double | L |
| temp | 随机游走: 80~100°C，每步±5°C | double | °C |
| turnSignal | 手动设置: 0=关, 1=左, 2=右 | int | - |
| hazard | 手动设置: false=关, true=开 | bool | - |

#### 2.1.3 随机故障报警

| 故障名 | 触发条件 | 持续时间 | 报警灯索引 |
|-------|---------|---------|-----------|
| absFault | 3%概率/秒，随机触发 | 5~15秒 | 7 (ABS) |
| airbagFault | 3%概率/秒，随机触发 | 5~15秒 | 6 (AIR安全气囊) |
| batteryFault | 3%概率/秒，随机触发 | 5~15秒 | 2 (BAT电瓶) |
| brakeFault | 3%概率/秒，随机触发 | 5~15秒 | 5 (BRAKE刹车) |
| speedFault | speed > 120 km/h时自动触发 | 速度低于120自动消除 | 0 (ENG超速报警) |

#### 2.1.4 控制指令

支持通过TCP接收以下控制指令（格式: `指令名:值`）：

| 指令 | 格式 | 说明 |
|-----|------|------|
| 转向灯 | `turnSignal:0` / `turnSignal:1` / `turnSignal:2` | 0=关, 1=左, 2=右 |
| 双闪 | `hazard:0` / `hazard:1` | 0=关, 1=开 |
| 重置 | `reset` | 重置所有状态到初始值 |

#### 2.1.5 JSON数据格式

模拟器发送的JSON示例：
```json
{
  "speed": 98.5,
  "rpm": 3940,
  "fuel": 59.45,
  "temp": 88.3,
  "turnSignal": 1,
  "hazard": 0,
  "absFault": 0,
  "airbagFault": 0,
  "batteryFault": 0,
  "brakeFault": 0,
  "speedFault": 0
}
```

### 2.2 仪表盘显示程序 (dashboard)

#### 2.2.1 窗口规格
- 窗口标题: "汽车仪表盘 - Socket 模式"
- 窗口大小: 900 x 650 像素
- 背景: 加载 `images/background.png`，若加载失败使用灰色背景(#c0c0c0)

#### 2.2.2 仪表组件布局

```
┌─────────────────────────────────────────────────────────┐
│  菜单栏: [信号控制] [报警灯测试]                          │
├───────────────┬───────────────────┬─────────────────────┤
│               │   [转向灯] ← →    │                     │
│               │                   │                     │
│   速度表       │   油量表  水温表   │      转速表         │
│  (Speedometer)│   (FuelGauge)    │    (RpmGauge)       │
│   左侧固定宽度  │   (TempGauge)    │    右侧固定宽度      │
│               │                   │                     │
│               │    [报警灯组]      │                     │
│               │ (WarningLights)   │                     │
└───────────────┴───────────────────┴─────────────────────┘
```

#### 2.2.3 速度表 (Speedometer)
- 显示范围: 0 ~ 240 km/h
- 刻度: 0, 20, 40, 60, 80, 100, 120, 140, 160, 180, 200, 220, 240
- 高速区(≥120)刻度颜色: 红色(#ff4400)
- 常规刻度颜色: 绿色(#00ffaa)
- 指针颜色: 红色(#ff2200)
- 数字颜色: 绿色(#00ff88)
- 数字字体: Arial, 36pt, Bold
- 动画: 支持平滑插值过渡（当前值向目标值以0.1系数逼近）

#### 2.2.4 转速表 (RpmGauge)
- 显示范围: 0 ~ 8000 RPM
- 刻度: 0, 1k, 2k, 3k, 4k, 5k, 6k, 7k, 8k
- 高速区(≥6000)刻度颜色: 红色
- 常规刻度颜色: 绿色

#### 2.2.5 油量表 (FuelGauge)
- 显示范围: 0 ~ 60 L
- 低油量报警: fuel < 15L 时自动触发 Warning_Fuel

#### 2.2.6 水温表 (TempGauge)
- 显示范围: 40 ~ 120 °C
- 高水温报警: temp > 95°C 时自动触发 Warning_Temp

#### 2.2.7 转向灯 (TurnIndicator)
- 两种状态: Indicator_Left（左）, Indicator_Right（右）
- 支持闪烁动画: 亮/暗交替，周期500ms
- 双闪(hazard)激活时: 左右转向灯同时闪烁

#### 2.2.8 报警灯组 (WarningLights)

8个报警灯，从左到右排列：

| 索引 | 枚举名 | 标签 | 触发条件 | 图片文件 |
|-----|--------|-----|---------|---------|
| 0 | Warning_Engine | ENG (超速报警) | speedFault (speed>120) 或菜单手动触发 | speed.png |
| 1 | Warning_Oil | OIL (机油) | 菜单手动触发 | oil.png |
| 2 | Warning_Battery | BAT (电瓶) | batteryFault 或菜单手动触发 | battery_fault.png |
| 3 | Warning_Temp | TEMP (水温) | temp > 95°C 或菜单手动触发 | temp.jpg |
| 4 | Warning_Fuel | FUEL (油量) | fuel < 15L 或菜单手动触发 | fuel.jpeg |
| 5 | Warning_Brake | BRAKE (刹车) | brakeFault 或菜单手动触发 | brake.png |
| 6 | Warning_Airbag | AIR (安全气囊) | airbagFault 或菜单手动触发 | air bag.png |
| 7 | Warning_ABS | ABS | absFault 或菜单手动触发 | abs.png |

#### 2.2.9 报警音效系统 (WarningSoundSystem)

支持报警灯触发时播放对应音效，音效文件存放在 `sounds/` 目录。

**音频格式**: WAV 或 MP3

**音效配置**:

| 音效ID | 文件名 | 触发条件 | 播放行为 |
|-------|--------|---------|---------|
| sound_engine | engine.wav | speedFault 触发（speed > 120 km/h） | 速度低于120自动停止 |
| sound_oil | oil.wav | 菜单手动触发 Warning_Oil | 单次播放 |
| sound_battery | battery.wav | batteryFault 触发 | 持续5~15秒（与故障同步） |
| sound_temp | temp.wav | temp > 95°C | 水温低于95°C自动停止 |
| sound_fuel | fuel.wav | fuel < 15L | 持续报警，直到加油 |
| sound_brake | brake.wav | brakeFault 触发 | 持续5~15秒（与故障同步） |
| sound_airbag | airbag.wav | airbagFault 触发 | 持续5~15秒（与故障同步） |
| sound_abs | abs.wav | absFault 触发 | 持续5~15秒（与故障同步） |

**功能特性**:
- **静音控制**: 菜单栏增加 `[静音]` 勾选框，勾选后禁用所有音效
- **音量控制**: 音量范围 0%~100%，默认 80%
- **单音播放**: 同一时刻只播放一个报警音效，新报警触发时替换正在播放的音效
- **停止机制**: 报警条件解除后，音效播放完毕当前循环后自动停止（不需要立即停止）

**菜单栏扩展**:

在现有菜单栏基础上增加音效控制菜单：


#### 2.2.10 菜单栏功能（修订）

原 2.2.9 菜单栏功能内容保持不变，调整编号为 2.2.10。

**信号控制菜单**:
- 左转: 设置 turnSignal:1
- 右转: 设置 turnSignal:2
- 双闪警告: 设置 hazard:1/hazard:0
- 互斥逻辑: 选择左转时自动取消右转，反之亦然

**报警灯测试菜单**:
- 8个可勾选菜单项，对应8个报警灯
- 点击可手动开启/关闭对应报警灯

#### 2.2.10 网络客户端 (NetworkClient)

- 连接目标: `127.0.0.1:5000`
- 自动重连: 连接断开后每3秒重试
- 连接超时: 5秒
- 解析接收的JSON并发出相应Qt信号

## 3. 数据通信协议

### 3.1 TCP连接

| 端 | 角色 | 端口 | 地址 |
|---|-----|------|------|
| final_simulator | 服务器 | 5000 | 0.0.0.0 (所有网卡) |
| dashboard | 客户端 | - | 127.0.0.1 |

### 3.2 数据流向

**模拟器 → 仪表盘（100ms周期）**:
```
{"speed":98.5,"rpm":3940,"fuel":59.45,"temp":88.3,"turnSignal":1,"hazard":0,"absFault":0,"airbagFault":0,"batteryFault":0,"brakeFault":0,"speedFault":0}
```

**仪表盘 → 模拟器（用户操作时）**:
```
turnSignal:1
turnSignal:2
turnSignal:0
hazard:1
hazard:0
reset
```

### 3.3 消息分隔
每条JSON消息以换行符 `\n` 分隔，支持一次接收多行。

## 6. 运行要求

### 6.1 启动顺序
1. **首先启动** `final_simulator`（TCP服务器，监听5000端口）
2. **然后启动** `dashboard`（TCP客户端，自动连接localhost:5000）

### 6.2 依赖
- Qt 5.x
- Qt Network 模块
- Qt Widgets 模块

### 6.3 图片资源
dashboard 程序目录下的 `images/` 文件夹必须包含所有报警灯图片，否则程序仍可运行但报警灯图标为空。
