## P6 服务端开发工程师 — 联调说明

| 项目 | 说明 |
|------|------|
| 客户端实现 | NetworkClient 已实现，使用 `--connect` 模式启动 |
| 通信协议 | JSON over TCP，端口 **5000**，消息以 `\n` 分隔 |
| 联调前准备 | 启动模拟器服务端，监听 5000 端口 |
| 启动命令 | `.\release\dashboard.exe --connect` |
| 联调步骤 | P6 启动模拟器 -> 确认端口可连接 -> 通知客户端发起连接 -> 验证数据收发 |
| 协议格式 | 每行一条JSON消息，包含字段名及对应数值，具体字段定义见接口文档 |

> 联调通过标准：仪表盘UI数据刷新与模拟器发送的数值一致，无断连或数据错乱。

## P7 测试工程师 — 测试说明

| 测试模式 | 说明 | 前置条件 |
|----------|------|----------|
| Mock 模式 | 双击 `run.bat` 或运行 `.\release\dashboard.exe`，使用内置Mock数据源 | 无需模拟器，可直接执行 |
| 网络模式 | 运行 `.\release\dashboard.exe --connect`，连接P6模拟器 | 需P6启动模拟器并监听5000端口 |

- 测试用例参考需求文档 **Section 7** 中的场景描述和预期结果。
- Mock 模式适用于功能验证、UI回归测试和自动化测试。
- 网络模式适用于联调测试、端到端场景验证。
- 发现缺陷请提交至项目缺陷跟踪系统，标注对应的测试模式和环境信息。
